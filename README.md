# 📘 Проект БСА: Пользовательские истории (User Stories)

Артефакты по выявлению, оформлению и согласованию пользовательских историй для ключевых ролей системы. Содержат описания US, критерии приёмки, тест-сценарии и протокол валидации.

## 📁 Структура файлов
| Упражнение | Имя файла | Что внутри |
|------------|-----------|------------|
| `ex00` | `ex00_<cut>_us_manager.docx` | 2 US для роли «Менеджер» + критерии приёмки (1 US) + тест-сценарий (1 US) |
| `ex01` | `ex01_<cut>_us_<роль>.docx` | ≥4 US для двух других ролей (по 2 на роль) + критерии приёмки и тест-сценарии |
| `ex02` | `ex02_<cut>_us.docx` | Протокол согласования: проверка 3C (Card, Conversation, Confirmation), INVEST, внесённые правки и причины |

## 🏷️ Правило именования
Формат: `exXX_<префикс продукта>_us_<роль>.docx`  
Примеры: `ex00_BSA01_us_manager.docx`, `ex01_BSA01_us_analyst.docx`

## 🧭 Порядок работы
1. `ex00` → базовые потребности и US менеджера.
2. `ex01` → расширение на другие роли (≥2 US на роль).
3. `ex02` → валидация с коллегами по 3C и INVEST, фиксация изменений в протоколе.
